Sistemas de información:

Conjunto de recursos físicos, humanos y de protocolo encaminados a gestionar y organizar información.

Componentes de un sistema de información:

  • Datos: Información relevante que se almacena y gestiona.
  • Hardware: Equipamiento físico que se emplea para gestionar los datos.
  • Software: Aplicaciones informáticas que se encargan de la gestión de la base de datos.
  • Recursos humanos: Personal que maneja el sistema de información.
Tipos de sistemas de información:
  1. Sistemas de información orientados al proceso
  2. Sistemas de información orientados a los datos

1 Sistemas de información orientados al proceso (sistemas de ficheros):

Emplean diversas aplicaciones. Cada aplicación almacena sus propios datos independientemente. La modificación de un dato en una aplicación no afecta al resto de aplicaciones.

Inconvenientes:
  • Datos redundantes.
  • Datos inconsistentes: Si cambias el dato en una aplicación, el mismo dato podría quedar con distinto valor en otra aplicación.
  • Coste de almacenamiento elevado: Repetición de datos.
  • Difícil acceso: Si se requiere una consulta no prevista, hay que cambiar el código o crear una nueva aplicación.
  • Tiempos de prosesamiento elevado: Almacenamiento no optimizado.
  • Dificultad de acceso simultaneo: Dos usuarios no pueden acceder a los datos de forma concurrente.
  • Dificultad para administrar la seguridad: cada aplicación se crea independientemente; siendo muy difícil establecer criterios de seguridad uniformes.

2 Sistemas de información orientados a los datos (bases de datos):

los datos se centralizan en una base de datos común, en una única estructura lógica que es utilizable por las aplicaciones. Cuando una aplicación modifica un dato, la modificación de dicho dato será visible para el resto de aplicaciones.

Ventajas:
  • Independencia de los datos, los programas y los procesos: Permite modificar los datos sin modificar el código de las aplicaciones.
  • No redundancia: Menor repetición de datos, solo se indican las relaciones.
  • Integridad de los datos: Mayor dificultad de perder relaciones entre los datos o de realizar incoherencias con ellos.
  • Datos más documentados: Metadatos.
  • Acceso a los datos más eficiente: La organización de los datos produce un resultado más optimizado en cuanto al rendimiento.
  • Menor espacio de almacenamiento: Gracias a la no redundancia y a la mejor estructuración de los datos.
  • Acceso simultáneo: Es más fácil controlar el acceso de usuarios de forma concurrente. Para evitar interferencias al modificar un mismo dato de forma concurrente, se usan transferencias (conjunto de operaciones simples que se ejecutan como una unidad, o se realizan todas, o no se realiza ninguna). Cuando dos transacciones se ejecutan de forma concurrente pueden darse interferencias, para evitarlo, mientras una transferencia actúa sobre unos datos, estos se bloquean para las demás transferencias generando esperas.
  • Mayor seguridad: Premite limitar el acceso a los usuarios. Cada tipo de usuario podrá acceder a unas cosas. Permite la generación de usuarios, contraseñas, encriptaciones, copias de respaldo seguras.
Desventajas:
  • Instalación costosa.
  • Personal cualificado.
  • Implantación larga y difícil.
  • Ausencia de estándares reales: dependencia hacia los sistemas comerciales del mercado. Esto ha mejorado significativamente ya que una gran parte de esta tecnología está aceptada como estándar.

Base de datos:

Colección de datos relacionados, estructurados y organizados entre sí, junto con el conjunto de programas que acceden y gestionan esos datos procurando seguridad, integridad y concurrencia.



Niveles de abstracción de una base de datos:

El modelo ANSI es una propuesta teórica sobre cómo debe funcionar un sistema gestor de bases de datos.

Según el modelo ANSI-SPARC, se utilizan tres esquemas para representar los datos:

  1. Esquema externo: Información preparada para el usuario final.
  2. Esquema conceptual: Refleja los datos y relaciones de la base de datos.
  3. Esquema interno: Representa la forma física en la cual están almacenados los datos.
